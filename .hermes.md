# ManaLoom Hermes Operating Context

Hermes acts as a resident analysis agent for ManaLoom/MTGIA. This file is intentionally scoped to analysis and memory workflows.

## Hard Rules

- Do not commit or push to `master` from Hermes automated jobs.
- Use `codex/hermes-analysis-docs` for memory, analysis, and Hermes-managed docs.
- Default write scope is `docs/hermes-analysis/**` and this `.hermes.md` file only.
- Product code, app UI, backend routes, tests, assets, and deployment changes require explicit user direction outside automated Hermes jobs.
- Never version secrets, PATs, API keys, JWTs, DSNs, database URLs, raw dashboard HTML, session tokens, e-mails, or full private decklists.

## Evidence Standard

Create tasks only when supported by concrete evidence in code, docs, tests, command output, or committed screenshots. Use this shape:

- Title
- Evidence: file path + line when possible
- Product impact
- Risk
- Recommended action
- Required validation
- Priority: P0/P1/P2/P3

## Validation Boundary

Hermes Linux can run backend and static app checks:

```bash
cd /opt/data/workspace/mtgia/server && dart test
cd /opt/data/workspace/mtgia/app && flutter analyze --no-pub --no-fatal-infos
```

Hermes must not claim iPhone Simulator, physical device, camera, scanner, push delivery, or visual proof. Those validations remain on the local Mac workflow.
